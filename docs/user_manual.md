# 仕入原価管理システム 総合手順書

この手順書は、このプロジェクトを初めて見る人でも「何のためのシステムか」「どう準備するか」「毎日どう使うか」が分かるようにまとめたものです。

## 1. このシステムの目的

このシステムは、店舗仕入れ・ネット仕入れの商品について、後から正確な仕入原価を確認できるようにするための記録システムです。

記録する主な内容は以下です。

- いつ仕入れたか
- どこで仕入れたか
- 何を仕入れたか
- いくらで仕入れたか
- 商品写真
- レシート写真

売れた後に「この商品の仕入値はいくらだったか」が分からなくなる問題を防ぐことが目的です。

## 2. 解決したい問題

このシステムは、以下のような困りごとを解決するために作られています。

- 売れた時に仕入値が分からない
- 同じ商品でも仕入れるたびに価格が違う
- レシートに商品名が出ない店舗がある
- クレジットカード明細だけでは商品を特定できない
- 商品写真、レシート、メモが別々に散らばる
- 後から原価確認をしようとしても根拠が残っていない

重要なのは、レシートから商品名を完全自動判定することではありません。仕入れた瞬間に、最低限の原価情報を残すことです。

## 3. 全体構成

このシステムは、Googleの標準機能を中心に作られています。

| 役割 | 使うもの | 内容 |
|---|---|---|
| 入力画面 | Googleフォーム | スマホから仕入内容を登録する |
| データ保存 | Googleスプレッドシート | 仕入DB、未確認リスト、集計を管理する |
| 写真保存 | Googleドライブ | 商品写真、レシート写真を保存する |
| 自動処理 | Google Apps Script | フォーム回答を整形して仕入DBへ転記する |
| 開発管理 | GitHub | コード、仕様書、手順書を保存する |

## 5. 利用者が使う主な画面

通常の運用で使う画面は2つです。

### 5.1 フォーム入力URL

仕入れを登録するためのGoogleフォームです。

スマホで開いて、仕入れた直後またはその日の最後に入力します。

### 5.2 仕入DB URL

登録された内容を見るためのGoogleスプレッドシートです。

後から内容を確認したり、商品名や単価を補完したり、日別・店舗別・支払方法別の集計を見るために使います。

## 6. 初回セットアップ

初めて使う場合は、Google Apps Scriptにコードを貼り付けて、フォームとスプレッドシートを作成します。

### 6.1 Apps Scriptプロジェクトを作る

1. Google Driveを開きます。
2. 「新規」を押します。
3. 「その他」を選びます。
4. 「Google Apps Script」を開きます。

### 6.2 `Code.gs` を貼り付ける

1. Apps Script画面で `Code.gs` を開きます。
2. 既存の内容をすべて削除します。
3. このフォルダの `apps-script/Code.gs` の内容をすべてコピーします。
4. Apps Script側の `Code.gs` に貼り付けます。
5. 保存します。

最後の行が以下のようになっていれば、最後まで貼り付けできています。

```js
function zeroPad_(value, length) {
  var text = String(value);
  while (text.length < length) {
    text = '0' + text;
  }
  return text;
}
```

### 6.3 `appsscript.json` を貼り付ける

1. Apps Script左側の「プロジェクトの設定」を開きます。
2. 「appsscript.json マニフェスト ファイルをエディタで表示する」をオンにします。
3. エディタに戻ります。
4. `appsscript.json` を開きます。
5. このフォルダの `apps-script/appsscript.json` の内容で置き換えます。
6. 保存します。

### 6.4 セットアップ関数を実行する

1. Apps Script上部の関数選択で `setupPurchaseCostSystem` を選びます。
2. 「実行」を押します。
3. 初回は権限確認が出るので許可します。
4. 実行ログを確認します。

成功すると、ログに以下のようなURLが表示されます。

```text
仕入DB: https://docs.google.com/spreadsheets/...
フォーム編集URL: https://docs.google.com/forms/...
フォーム入力URL: https://docs.google.com/forms/...
```

## 7. フォームの写真項目を設定する

Apps Scriptの標準機能では、環境によってファイルアップロード項目を自動作成できない場合があります。

その場合、`商品写真` と `レシート写真` は一時的にURL入力欄として作られます。実運用前にGoogleフォーム編集画面で、以下の2項目をファイルアップロードに変更してください。

### 7.1 商品写真

1. フォーム編集画面を開きます。
2. `商品写真` の質問を削除します。
3. 新しい質問を追加します。
4. 質問タイトルを `商品写真` にします。
5. 質問タイプを `ファイルのアップロード` にします。
6. 必須にします。

### 7.2 レシート写真

1. `レシート写真` の質問を削除します。
2. 新しい質問を追加します。
3. 質問タイトルを `レシート写真` にします。
4. 質問タイプを `ファイルのアップロード` にします。
5. 必須にします。

質問タイトルは必ず `商品写真`、`レシート写真` のままにしてください。Apps Scriptがこの名前を使って回答を読み取ります。

## 8. フォーム入力方法

スマホから30秒以内で登録できることを優先します。

### 8.1 仕入れた直後に登録する場合

1. スマホで `フォーム入力URL` を開きます。
2. 店舗名を入力します。
3. 合計金額を入力します。
4. 商品名メモを入力します。
5. 商品写真を添付します。
6. レシート写真を添付します。
7. 必要なら支払方法、数量、型番、JAN、ASIN、備考を入力します。
8. 送信します。

商品名メモは雑で構いません。

例:

```text
switch lite グレー 本体のみ
ps4 コントローラー 黒 2個
ポケモン ソフト まとめ
```

音声入力を使っても問題ありません。

### 8.2 その日の最後にまとめて登録する場合

登録忘れがあった場合は、その日の最後にまとめて入力しても構いません。

その場合は、仕入日を実際に仕入れた日に合わせてください。仕入日を空欄にすると、登録日が仕入日として扱われます。

## 9. 入力項目の意味

| 項目 | 必須 | 内容 |
|---|---|---|
| 仕入日 | 任意 | 実際に仕入れた日。空欄なら登録日扱い |
| 店舗名 | 必須 | GEO、ハードオフ、Amazonなど |
| 合計金額 | 必須 | レシートや注文の合計金額 |
| 商品名メモ | 必須 | 後で商品を思い出せるメモ |
| 商品写真 | 必須 | 商品を特定するための写真 |
| レシート写真 | 必須 | 金額や日付の根拠 |
| 支払方法 | 任意 | 現金、楽天カード、PayPayなど |
| 数量 | 任意 | 空欄なら1 |
| 型番 | 任意 | 分かる場合のみ |
| JAN | 任意 | 分かる場合のみ |
| ASIN | 任意 | Amazon商品なら分かる場合のみ |
| 備考 | 任意 | その他メモ |

## 10. スプレッドシートの見方

仕入DB URLを開くと、以下のシートがあります。

### 10.1 仕入DB

すべての仕入データが入るメインのシートです。

フォーム送信後、Apps Scriptが自動で1行追加します。

主な列は以下です。

| 列 | 内容 |
|---|---|
| 仕入ID | 自動採番される管理番号 |
| 登録日時 | フォームを送信した日時 |
| 仕入日 | 実際の仕入日 |
| 店舗名 | 仕入先 |
| 支払方法 | 支払い方法 |
| 合計金額 | レシートや注文の合計 |
| 商品名メモ | フォームで入力したメモ |
| 商品名_確定 | 後で確定した正式な商品名 |
| 型番 | 任意 |
| JAN | 任意 |
| ASIN | 任意 |
| 数量 | 基本は1 |
| 単価 | 後で入力する商品ごとの原価 |
| 商品写真URL | Google Drive上の商品写真 |
| レシート写真URL | Google Drive上のレシート写真 |
| ステータス | 未確認、確認済、売却済、除外 |

### 10.2 未確認リスト

確認や補完が必要な仕入だけを表示します。

以下に当てはまる行が表示されます。

- ステータスが `未確認`
- 商品名_確定 が空欄
- 数量 が空欄
- 単価 が空欄
- 商品写真URL が空欄
- レシート写真URL が空欄

日々の確認作業では、まずこのシートを見ます。

### 10.3 日別集計

仕入日ごとの件数と合計金額を確認できます。

「今日はいくら仕入れたか」「今月どのくらい仕入れたか」を見るために使います。

### 10.4 店舗別集計

店舗ごとの仕入件数と合計金額を確認できます。

「どの店舗で多く仕入れているか」を見るために使います。

### 10.5 支払方法別集計

支払方法ごとの件数と合計金額を確認できます。

カード明細や現金管理と照らし合わせる時に使います。

### 10.6 設定

フォーム入力URL、フォーム編集URL、仕入DB URLが保存されます。

URLが分からなくなった時は、このシートを確認してください。

## 11. 毎日の運用

毎日の基本運用は以下です。

1. 仕入れたらスマホでフォームを開きます。
2. 店舗名、合計金額、商品名メモを入力します。
3. 商品写真とレシート写真を添付します。
4. 送信します。
5. その日の最後に仕入DB URLを開きます。
6. `未確認リスト` を見ます。
7. 商品名_確定、単価、数量などを補完します。
8. 確認できたら `仕入DB` のステータスを `確認済` にします。

## 12. ステータスの使い方

| ステータス | 意味 |
|---|---|
| 未確認 | フォーム登録直後。まだ内容確認が必要 |
| 確認済 | 商品名、金額、数量などを確認済み |
| 売却済 | 売上データと紐付け済み |
| 除外 | 私用、経費、誤登録などで原価管理から外す |

最初は必ず `未確認` で登録されます。

確認作業が終わったら、手動で `確認済` に変更します。

## 13. 仕入IDのルール

仕入IDは自動で発行されます。

形式は以下です。

```text
P-YYYYMMDD-連番
```

例:

```text
P-20260614-001
P-20260614-002
P-20260615-001
```

日付ごとに連番が振られます。

## 14. よくある作業例

### 14.1 GEOでゲーム機を仕入れた場合

フォームに以下のように入力します。

| 項目 | 入力例 |
|---|---|
| 店舗名 | GEO |
| 合計金額 | 15000 |
| 商品名メモ | switch lite グレー 本体のみ |
| 商品写真 | 商品写真を添付 |
| レシート写真 | レシート写真を添付 |
| 支払方法 | 楽天カード |

後で未確認リストを開き、商品名_確定に以下のように入れます。

```text
Nintendo Switch Lite グレー
```

単価に `15000`、数量に `1` を入れ、問題なければステータスを `確認済` にします。

### 14.2 複数商品をまとめて仕入れた場合

第1段階では、まずレシート単位で登録します。

商品ごとの単価が後から必要な場合は、未確認リストで確認しながら単価を補完します。

将来的には商品マスタや明細分割に対応する予定ですが、今は「仕入れた情報を残すこと」を優先します。

## 15. やってはいけないこと

- レシートだけで商品を特定しようとしない
- 商品写真を撮らずに登録しない
- 商品名メモを空欄にしない
- 既存の仕入DB行を安易に削除しない
- AI解析やOCRだけに頼らない
- 最初から複雑な運用にしない

削除が必要な場合は、まずステータスを `除外` にする運用を推奨します。

## 16. トラブル対応

### 16.1 Apps Scriptで「Unexpected end of input」が出る

`Code.gs` の貼り付けが途中で切れています。

`apps-script/Code.gs` をもう一度すべてコピーし、Apps Script側の `Code.gs` に貼り直してください。

最後が以下で終わっているか確認します。

```js
function zeroPad_(value, length) {
  var text = String(value);
  while (text.length < length) {
    text = '0' + text;
  }
  return text;
}
```

### 16.2 `form.addFileUploadItem is not a function` が出る

古いコードを使っている可能性があります。

修正版では、ファイルアップロード項目を自動作成できない場合でもフォーム作成が止まらないようになっています。

最新の `apps-script/Code.gs` を貼り直してください。

### 16.3 フォームがどこにあるか分からない

以下のどれかで探せます。

- Apps Scriptの実行ログ
- Google Driveの「最近使用したアイテム」
- スプレッドシートの `設定` シート

フォーム名は `仕入登録フォーム` です。

### 16.4 仕入DBがどこにあるか分からない

Apps Scriptの実行ログに表示される `仕入DB` のURLを開きます。

またはGoogle Driveで `仕入原価管理システム` を検索します。

### 16.5 フォーム送信しても仕入DBに反映されない

以下を確認してください。

1. Apps Scriptのトリガーに `onPurchaseFormSubmit` があるか
2. フォームの回答先が作成されたスプレッドシートになっているか
3. Apps Scriptの権限許可が完了しているか
4. 質問名 `商品写真`、`レシート写真` を変更していないか

### 16.6 写真URLが入らない

フォームの写真項目名が以下と完全一致しているか確認してください。

```text
商品写真
レシート写真
```

名前が少しでも違うと、Apps Scriptが写真欄を読み取れません。

## 20. 運用の基本方針

このシステムを続けるために、一番大事なのは入力を重くしないことです。

登録時点では、完璧な商品名や単価まで入れなくても構いません。

まずは以下だけ残してください。

- 店舗名
- 合計金額
- 商品名メモ
- 商品写真
- レシート写真

後から未確認リストで補完すれば大丈夫です。
