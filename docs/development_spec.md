# 仕入原価管理システム 開発仕様書

## 1. 目的

店舗仕入れ・ネット仕入れの商品について、仕入日・店舗・商品名・金額・画像・レシートを記録し、後日商品が売れた際に正確な仕入原価を確認できる状態を作る。

特に以下の問題を解決する。

- 売れた時に仕入値が分からない
- 同じ商品でも毎回仕入値が異なる
- レシートに商品名が出ない店舗がある
- クレジットカード明細だけでは商品が特定できない
- 仕入れた商品写真・レシート・メモがバラバラになる

## 2. 基本方針

第1段階では、完全自動化を狙わず、Googleフォームから仕入記録を残すことを最優先とする。

第2段階で、商品写真・レシート画像をAIで解析して補完する。

第3段階で、売上データと紐付けて利益計算に使う。

## 3. システム構成

- Googleフォーム
  - 仕入登録用
- Googleスプレッドシート
  - 仕入DB
  - 商品マスタ
  - 集計
  - 未確認リスト
- Googleドライブ
  - 商品写真・レシート写真保存
- Google Apps Script
  - フォーム送信時の整形
  - 画像URL保存
  - 管理番号発行
  - 未入力チェック
- 将来的にAI API
  - 商品画像解析
  - レシートOCR
  - 商品名・型番・JAN補完

## 4. Googleフォーム入力項目

### 必須項目

- 仕入日
- 店舗名
- 支払方法
- 合計金額
- 商品名メモ
- 商品写真
- レシート写真

### 任意項目

- 商品状態
- 数量
- 型番
- JAN
- ASIN
- 楽天商品管理番号
- 備考
- 販売予定先
- 仕入担当者

## 5. スプレッドシート列設計

### 仕入DB

| 列名 | 内容 |
|---|---|
| 仕入ID | 自動採番 |
| 登録日時 | フォーム送信日時 |
| 仕入日 | 実際の仕入日 |
| 店舗名 | GEO、ハードオフ、お宝、Amazon等 |
| 支払方法 | 楽天カード、PayPayカード、現金等 |
| 合計金額 | レシート合計 |
| 商品名メモ | 手入力メモ |
| 商品名_確定 | 後で確定する正式名称 |
| 型番 | 任意 |
| JAN | 任意 |
| ASIN | 任意 |
| 数量 | 基本1、複数時は入力 |
| 単価 | 商品ごとの仕入単価 |
| 送料 | 任意 |
| ポイント利用 | 任意 |
| 実質原価 | 単価＋送料−ポイント等 |
| 商品写真URL | Drive URL |
| レシート写真URL | Drive URL |
| ステータス | 未確認、確認済、売却済、除外 |
| 売上紐付けID | 後で利用 |
| メモ | 任意 |

## 6. ステータス定義

- 未確認
  - フォーム登録直後
- 確認済
  - 商品名・金額・数量が確認済
- 売却済
  - 売上データと紐付け済
- 除外
  - 私用、経費、誤登録など

## 7. 第1段階で作る機能

### 7.1 Googleフォーム作成

スマホからすぐ登録できるフォームを作成する。

最低限、以下を登録できること。

- 店舗名
- 支払方法
- 合計金額
- 商品名メモ
- 商品写真
- レシート写真

### 7.2 フォーム回答シート整形

フォーム送信後、Apps Scriptで仕入DBへ転記する。

処理内容：

- 仕入IDを自動発行
- 日付形式を統一
- 金額を数値化
- 画像URLを保存
- ステータスを「未確認」にする

### 7.3 未確認リスト作成

商品名_確定、単価、数量、画像が不足しているものを抽出する。

### 7.4 日別集計

日別で以下を集計する。

- 仕入件数
- 仕入合計金額
- 店舗別金額
- 支払方法別金額

## 8. 第2段階で作る機能

### 8.1 レシートOCR

レシート画像から以下を抽出する。

- 店舗名
- 日付
- 合計金額
- 商品行
- 消費税
- 支払方法

ただし、お宝・リサイクルショップ系は商品名が出ない場合があるため、商品特定の主情報にはしない。

### 8.2 商品写真AI解析

商品写真から以下を推定する。

- 商品名
- 型番
- 色
- 状態
- JAN候補
- ASIN候補

### 8.3 AI補完結果の確認欄

AIが出した商品名は自動確定せず、確認欄に出す。

人間が確認後、「商品名_確定」に反映する。

## 9. 第3段階で作る機能

### 9.1 売上データとの紐付け

楽天・Amazon・メルカリ等で売れた商品と仕入DBを照合する。

照合キー候補：

- JAN
- ASIN
- 商品名
- 型番
- 楽天商品管理番号
- 手動指定

### 9.2 原価候補表示

売れた商品に対して、過去の仕入履歴から原価候補を表示する。

表示例：

| 仕入日 | 店舗 | 商品名 | 原価 | ステータス |
|---|---|---|---|---|
| 2026/06/01 | GEO | Switch Lite グレー | 15,000 | 確認済 |
| 2026/06/08 | お宝 | Switch Lite グレー | 16,500 | 確認済 |

### 9.3 原価計算方式

初期は手動選択を基本とする。

将来的に以下を選べるようにする。

- 最新原価
- 平均原価
- 先入先出
- 手動選択

## 10. 重要な運用ルール

- レシートだけで商品特定しようとしない
- 商品写真は必ず撮る
- 商品名メモは雑でもよいので必ず入れる
- 店舗名と金額だけでも当日中に登録する
- AI補完は確認前提とする
- 原価確定前の商品は「未確認」に残す

## 11. 受け入れ条件

第1段階の完成条件：

- スマホからGoogleフォームで仕入登録できる
- 登録内容が仕入DBへ自動転記される
- 仕入IDが自動発行される
- 商品写真・レシート写真のURLが保存される
- 未確認リストが作成される
- 日別・店舗別・支払方法別の集計が見られる

第2段階の完成条件：

- レシート画像から店舗名・日付・合計金額を抽出できる
- 商品写真から商品名候補を出せる
- AI補完結果を人間が確認して確定できる

第3段階の完成条件：

- 売上データから該当商品の原価候補を表示できる
- 原価を選択して利益計算に使える
